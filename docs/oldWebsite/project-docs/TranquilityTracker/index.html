Overview

This mobile interface is a journal is a for tracking your levels of mind. Open the web-based drawing pad, pick a color, thickness and draw a wave that resembles your current level of tranquility, however calm or anxious that may be. The visual page (inside the info menu) allows you to interpret your mindset and inspect the patterns of your tension over the past month. A better understanding of the mental patterns in your day-to-day will allow you to live a more positive life. This project is MOBILE INTERACTION ONLY.

Initial Experiments

We did not know if this type of anxiety tracking would be insightful, so we began this project by tracking our tranquility with a simple sketching app. We settled on the iPhone notes app as our initial drawing pad.

Our initial experiments were excellent and we could quickly understand our “signals” with increasing accuracy and meaning. This meant that as you track your mood, you start to build your own mental scales and legends. We wanted to keep the expressiveness of drawing in the experiment so we did not go with a strict meaning to your signal (see initial concept image below.)

Drawing Pad Prototype

The goal was to create a quick as possible record of your current tranquility level. Thus, I wanted to create an interface where you don’t think too much about the pen choices and drawing choices. Once you choose your color and pen width you must continue with those choices (This is to take mental load off of the user as they attempt a difficult problem; representing your anxiety with one line.) See my initial mockups for the drawing interface below.

I thought the initial choice system delivered in the ways that I wanted; quick and not too many options. But there were multiple issues. First, the colors were horrible for representing anxiety. Second, the pen sizes didn’t equate to the actual drawing width as well as I would have liked (I later found out that implementing this sample line, in the top right figure, was super difficult and was taking more time than I wanted to spend on this feature.) The first running version was identical to these mockups, but without the control buttons, instead the line would automatically be sent off and then the drawing pad would erase.

Implementation
& Infrastructure

This project consists of two parts. The first part is the drawing pad application. This is a statically hosted web page that currently resides at [github-drawing link]. Once I edit that file to iterate on the pad, I call a deploy that copies the new iteration over to the specific directories for each test user (Casey, Dom, Clark, Noopur.) I chose this dev strategy because it opened me up to focus on the user flow and interface instead of scalability.


The next portion of this project was the backend server that collects data from the front-end drawing pad application. Since I am a student and did not want to run up and bills on AWS of GCP, I chose to use my own raspberry pi. I connected my pi to my home router and used port forwarding to direct POST and GET requests to my raspberry pi. I used node and npm to build a small server for accepting these requests. On a post request from the page, my server saves the line data into a sqlite database. On receiving a GET request, the page will render your calendar view on the server side using d3 in a node environment. This page gets returned with some extra javascript and d3 that allows for more interactivity. The code for the express server is here.
